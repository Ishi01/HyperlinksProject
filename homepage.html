Introduction

 Hyperlinks are one of the core concepts that allow the internet as we know it today to function. In the simplest terms, the function of a hyperlink is to redirect the user from where they currently are on a webpage to somewhere else, either somewhere in the same page, a different page on the same website, a different website entirely, or pretty anywhere you can go on the internet. The hyperlink can also direct to a specific section of the same, or another, webpage using an anchor tag. It can also initiate the download for an external resource or file, or provide an expanded view of an image or video.


Implementation of Hyperlinks (Web Developers)

 From the web-developers perspective, the implementation of hyperlinks varies depending on the purpose of the link. For on-page links, the implementation relies primarily on the use of an anchor tag. These tags allow you specify an href attribute, which is usually the ID of a page element which you want the user to jump to when the anchor tag is clicked. Since href attributes can be attached to virtually any element in a web page, this allows automatic scrolling to pretty much any specified element when the element is clicked. Additionally, anchor tags can be attached to virtually any element in a web page, but are most commonly attached to either buttons or links, using the classic blue underlined text, which has become the ubiquitous UI language for pretty much any form of link. For an example of this, see the index to the left of this text.

 When a link is used to direct a user to a different page on the same website, the most common implementation is to use some simple Javascript alongside the HTML in order to instruct the web browser to load a separate page. More precisely, this involves the usage of an onclick event handler for a specific HTML element (most commonly a link or button), which is used to call a javascript function. This function can either be placed in the same HTML document using the script tag, or in a separate javascript file linked to the HTML file. These functions redirect the page using a location.href change, using a location reference to a separate document in the website's document tree. For an example of this behavior, see the navbar at the top of this page, which uses this implementation to redirect to the other pages on this website.

 When the web developer wishes to use an element to redirect to a separate web page entirely, a slightly different implementation is required. The code on the web developers side is relatively similar, simply changing the location.href argument used in the previous section with the URL (Uniform Resource Locator) of the website you wish to redirect to. Alternatively, a simple anchor tag can be used to redirect to a different page by specifiying the URL in the href attribute. However, this changes what the web browser is required to do rather dramatically. When the web browser receives a location change request which redirects to a different web page, it necessitates the usage of a DNS Lookup in order to determine the IP address of the page you wish to load. The web browser then uses this IP address to load the page. More information on this will be provided in the Browsers section of the implementation. For more information on URLs (and an example of how this works), please follow the link present in the above paragraph.


Implementation of Hyperlinks (Web Browsers)

 The browser implementation of hyperlinks occurs in several stages. Firstly, the browser must retrieve and render the web page which contains the hyperlink. This is done through interpreting the HTML and CSS contained within the page, which can vary slightly depending on the browser. It then searches through the HTML and javascript to locate clickable elements, either anchor tags or other elements connected to javascript functions, and renders these accordingly. The browser then waits for the user to interact with these elements through the usage of Event Listeners, most commonly the onclick event listener mentioned above.

 Once one of these elements is interacted with/called, the browser either runs the associated javascript, which likely contains a reassignment of the location.href attribute, or searches the anchor tag for the href link. In both cases, these items contain the URL, or Uniform Resource Locator, of the website the developer wishes to link to. Once the browser determines this, it performs a DNS lookup of the URL. For this, the browser reaches out to a DNS server using the internet connection, and requests the IP address of the server which is associated with the URL. DNS servers are essentially a publicly available catalog of URLs and their associated IP addresses, which are used by browsers to translate the much more human readable URL into an IP address, which is what computers actually use to navigate the web.

 Now that the browser has the IP address of the server hosting the webpage it is trying to navigate to, it uses this IP to locate the server and perform an HTTP request, essentially asking the server to send it all of the HTML, CSS and Javascript it requires in order to render the webpage. It then downloads all of this content (and an associated multimedia, such as pictures or videos), and renders it according to the HTML and CSS the server provided. This page is then presented to the user, who can then interact with the page as they see fit. Notably, most browsers implement some form of a back button, which allows the user to return to the page they were previously on. These pages are often also cached, to limit the amount of data the browser needs to download in order to render the page, as well as the loading times.
